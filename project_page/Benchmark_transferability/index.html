RobustSSL Benchmark


Introduction to Robust Self-Supervised Learning (RobustSSL) Benchmark

The wide-ranging applications of foundation models, espeically in safety-critical areas, necessitates the robust self-supervised learning (RobustSSL) which can yield adversarially robust foundation models. Fine-tuning these robust foundation models can lead to strong adversarial robustness in downstream tasks. Here, we build a leaderboard for transferability of RobustSSL methods, which aims to provide a comprehensive evaluation of RobustSSL methods and facilitate the research in robust pre-training.

Performance Mesurement

The code for conducting fine-tuning and evaluation is provided in GitHub.

Fine-Tuning Modes

  • Standard linear fine-tuning (SLF): only standardly fine-tuning the classifier while freezing the encoder.
  • Adversarial linear fine-tuning (ALF): only adversarially fine-tuning the classifier while freezing the encoder.
  • Adversarial full fine-tuning (AFF): adversarially fine-tuning both the encoder and the classifier.

Evaluation Metrics

  • Standard accuracy refers to the standard test accuracy evaluated on the natural test data.
  • Robust accuracy refers to the robust test accuracy evaluated on the adversarial test data generated via the standard version of AutoAttack.
  • Corruption accuracy refers to the mean test accuracy of the test data under common corruptions with corruption severity ranging {1,2,3,4,5}.

Leaderboards

  • Transferability from CIFAR-10 to CIFAR-10
  • Transferability from CIFAR-100 to CIFAR-100
  • Transferability from STL-10 to STL-10
  • Transferability from CIFAR-10 to STL-10
  • Transferability from CIFAR-100 to STL-10
